## 3. 模块详解

### 3.1 程序入口 (`main.py`)

启动流程：
1. 加载 `.env` 环境变量
2. 初始化日志系统 (`RotatingFileHandler`)
3. 初始化 SQLite 数据库，若首次运行创建默认账户
4. 初始化 PolymarketAPI（含代理配置）
5. 搜索并定位当前活跃的 BTC 价格预测市场
6. 初始化策略管理器，同步策略到数据库
7. 初始化账户管理器
8. 初始化交易模拟器
9. 首次获取市场数据
10. 启动调度器后台线程（每 `fetch_interval` 秒循环获取数据+执行策略）
11. 启动 Flask Web 服务

### 3.2 API 层 (`src/api/polymarket_api.py`)

封装 Polymarket 三个 API 端点：

| API | 地址 | 功能 |
|-----|------|------|
| Gamma API | `gamma-api.polymarket.com` | 市场搜索、市场详情 |
| CLOB API | `clob.polymarket.com` | 实时价格、交易簿 |
| Data API | `data-api.polymarket.com` | K线历史价格 |

**代理支持**：通过 `.env` 的 `PROXY_URL` 配置。自动将 `socks5://` 转为 `socks5h://`（远程DNS解析），解决 SOCKS5 代理下 HTTPS SSL 错误。

**速率限制**：内置请求间隔（≥1秒），防止被 API 限制。

主要方法：
- `search_btc_markets(limit)` — 多维度搜索 BTC 市场
- `get_market_by_slug(slug)` — 获取市场详情（含 tokens）
- `get_price_history(token_id, interval, limit)` — 获取K线历史
- `get_realtime_price(token_id)` — 获取实时买卖价和中间价

### 3.3 数据库层 (`src/database/db_manager.py`)

SQLite 数据库（WAL 模式，外键约束，线程安全）。

#### 表结构

**`accounts` — 账户表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 账户ID |

| balance | REAL | 当前可用余额 |
| position | REAL | 持仓量 |

| created_at / updated_at | INTEGER | 时间戳 |

**`strategies` — 策略注册表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 策略ID |

| display_name | TEXT | 显示名称 |
| class_name | TEXT | Python 类名 |

| default_params | TEXT | 默认参数 JSON |

**`account_strategy` — 账户策略绑定表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 绑定ID |
| account_id | FK→accounts | 账户ID |
| strategy_id | FK→strategies | 策略ID |
| enabled | INTEGER | 启用标志 |

| 字段 | 类型 | 说明 |
|------|------|------|

| timestamp | INTEGER | K线时间 |

**`trades` — 交易记录**

| 字段 | 类型 | 说明 |
|------|------|------|

### 3.4 策略系统

#### 架构

```
┌─────────────────────────┐
│  strategy_loader.py      │  自动扫描 plugins/ 目录
│  发现继承BaseStrategy的类  │
└───────────┬─────────────┘
            │ 注册到全局 _strategy_registry
            ▼
┌─────────────────────────┐
│  strategy_manager.py     │  sync_strategies_to_db()
│  同步到 strategies 表     │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│  Web API                 │  POST /api/account_strategy
│  绑定账户→策略            │  写入 account_strategy 表
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│  调度器 → run_all_enabled │  遍历已启用绑定
│  生成信号→执行交易         │
└─────────────────────────┘
```

#### 策略基类 (`BaseStrategy`)

```python
class BaseStrategy:
    name: str           # 策略唯一标识
    display_name: str   # 显示名称
    description: str    # 策略描述
    default_params: dict  # 默认参数
    params: dict        # 运行时参数

    def generate_signal(self, df: pd.DataFrame) -> dict:
        """返回 {"action": "BUY|SELL|HOLD", "confidence": float, "reason": str}"""

    # 内置指标工具
    calc_ema(data, period)     # 指数移动平均
    calc_sma(data, period)     # 简单移动平均
    calc_macd(data, fast, slow, signal)  # MACD
    calc_rsi(data, period)     # RSI
```

#### MACD 策略 (`MACDStrategy`)

- 参数：`fast=12, slow=26, signal=9, trend_filter=True, trend_ma_period=50`
- 逻辑：MACD 线（DIF）上穿信号线（DEA）→ BUY；下穿 → SELL
- 趋势过滤：价格必须高于 MA50 才允许买入，防止逆势交易

#### 双均线策略 (`MAStrategy`)

- 参数：`short_period=20, long_period=60, ma_type="EMA", min_distance=0.0`
- 逻辑：快线上穿慢线 → BUY；下穿 → SELL
- 距离过滤：`min_distance` 设置最小均线距离，防止频繁交易

#### 扩展策略

只需 3 步：
1. 在 `src/strategy/plugins/` 新建 `.py` 文件
2. 继承 `BaseStrategy`，定义 `name / display_name / default_params`
3. 实现 `generate_signal(df)` 方法

### 3.5 交易模拟器 (`simulator.py`)

模拟账户买卖流程：

```
buy(account_id, strategy_id, price, reason)
  ├── 检查账户余额 >= value
  ├── 计算买入数量 = value / price
  ├── 更新持仓均价（加权平均）
  ├── 扣除余额
  ├── 记录 trade（action=BUY）
  └── 同步账户到数据库

sell(account_id, strategy_id, price, reason)
  ├── 检查持仓 > 0
  ├── 计算盈亏 = (price - avg_cost) * position
  ├── 归还余额 = price * position
  ├── 清零持仓
  ├── 记录 trade（action=SELL, 含pnl）
  └── 同步账户到数据库
```

### 3.6 账户管理 (`account.py`)

```
Account (单个账户)
├── account_id, name
├── initial_balance, balance
├── position (持仓量), avg_cost (持仓均价)
├── order_amount (每次下单金额)
├── buy_amount(position) → 买入开仓/加仓
├── sell_amount(position) → 卖出平仓
└── calculate_pnl(price) → 计算未实现盈亏

AccountManager (多账户管理器)
├── load_accounts() → 从DB加载到内存
├── get_account(id) → 获取账户
├── get_all_accounts() → 所有账户
└── _accounts: dict[id → Account]
```

### 3.7 Web 服务 (`src/web/app.py`)

Flask 应用，提供 1 个页面 + 10 个 REST API 端点。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | Dashboard 主页 |
| GET | `/api/status` | 系统状态 |
| GET/POST | `/api/accounts` | 账户列表 / 创建账户 |
| GET | `/api/accounts/<id>` | 账户详情 |
| GET | `/api/strategies` | 策略列表 |
| POST | `/api/account_strategy` | 绑定策略 |
| PUT/DELETE | `/api/account_strategy/<id>` | 更新/解绑 |
| GET | `/api/account_strategy/status` | 策略运行状态 |
| GET | `/api/prices` | K线价格数据 |
| GET | `/api/trades` | 交易历史 |
| GET | `/api/logs` | 系统日志 |

### 3.8 前端 Dashboard (`index.html`)

单页应用，5 秒自动轮询更新。

- **顶部栏**：状态指示灯、BTC 预测价格、市场链接
- **左侧**：K线图（Chart.js），可选 MA20/MA60 叠加
- **右侧**：
  - 账户统计卡片（多账户切换）
  - 策略管理面板（绑定/启用/禁用）
  - 交易历史表格
  - 系统日志面板
