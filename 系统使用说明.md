# 系统使用说明 — Polymarket Jack BTC

## 1. 启动系统

```bash
# 本地启动
python main.py

# 看到以下日志表示启动成功：
# [INFO] Polymarket Jack BTC 套利模拟程序 启动
# [INFO] 数据库初始化完成: data/trades.db
# [INFO] BTC市场: bitcoin-above-100k-...
# [INFO] Web Dashboard 启动: http://0.0.0.0:5000
```

打开浏览器访问 `http://127.0.0.1:5000`。

## 3. 账户管理

### 3.1 创建账户

1. 点击账户标签旁的 `+ 新建` 按钮
2. 填写表单：

| 字段 | 说明 | 示例 |
|------|------|------|
| 账户名称 | 唯一标识 | `激进账户` |
| 初始金额 (USDC) | 账户初始余额 | `1000` |
| 下单金额 (USDC) | 每次交易使用的金额 | `100` |

3. 点击"创建"确认

> **下单金额说明**：策略产生买入信号时，会使用该金额买入 YES 代币。仓位管理：100 USDC 买 63.5% 价格的代币 → 获得约 157 份份额。

### 3.2 切换账户

点击账户标签即切换，统计数据、策略列表、交易历史都会自动更新为该账户的数据。

## 4. 策略管理

### 4.1 绑定策略

1. 在策略管理区点击 `+ 绑定策略`
2. 下拉选择要绑定的策略（MACD 趋势跟踪 / 双均线趋势跟踪）
3. （可选）设置"分配资金" — 限制该策略的最大可使用资金
4. 点击确认

新绑定的策略默认**启用**状态。

### 4.2 启用/禁用策略

- 点击策略旁的开关按钮即可
- **启用**：该策略在当前账户上生效，调度器每轮会执行
- **禁用**：暂停该策略，不再产生交易信号

### 4.3 查看策略信号

策略列表中的信号指示灯显示最近一次信号：
- **BUY**（绿色）：策略产出买入信号
- **SELL**（红色）：策略产出卖出信号
- **HOLD**（灰色）：无操作信号

## 5. 配置调整

### 5.1 修改配置文件

编辑 `config.py`：

```python
# 数据获取间隔（秒），建议不小于30
BTC_MARKET = {
    "fetch_interval": 60,  # 每60秒获取一次数据并执行策略
}

# 账户默认值
ACCOUNT_DEFAULTS = {
    "initial_balance": 1000.0,   # 新账户默认初始金额
    "trade_amount": 50.0,        # 默认下单金额（可在创建时覆盖）
}

# Web 服务
WEB = {
    "port": 5000,     # 可改为其他端口
}
```

### 5.2 策略参数

策略参数在 `config.py` 中设置默认值，但可以通过 Web 绑定策略时自定义：

```python
STRATEGY_DEFAULTS = {
    "MACD": {
        "fast_period": 12,
        "slow_period": 26,
        "signal_period": 9,
        "trend_filter": True,
    },
    "MA": {
        "short_period": 20,
        "long_period": 60,
        "ma_type": "EMA",   # "EMA" 或 "SMA"
        "min_distance": 0.0,
    }
}
```

## 6. 常见问题

### Q: 启动后找不到 BTC 市场？

程序会自动搜索 Polymarket 上的 BTC 价格预测市场。如果市场已过期或没有活跃的 BTC 市场，日志会显示警告并使用默认市场。检查 Polkamarket 官网是否还有活跃的 BTC 相关市场。

### Q: 代理连接失败？

编辑 `.env` 文件，确认 `PROXY_URL` 正确。海外服务器无需代理，留空即可。

```
PROXY_URL=socks5://127.0.0.1:5001
```

### Q: 数据库文件损坏？

删除 `data/trades.db` 重新运行程序，会自动创建新数据库。注意：这会清空所有交易记录和账户数据。

### Q: 图表不显示数据？

确保程序已运行至少一轮数据获取（最多等待 fetch_interval 秒）。检查日志中是否有 "新增 X 条K线数据" 的记录。
